# 论文与系列文章选题

## 方向一：面向家庭物联网的多层网关接入与统一状态模型

重点：Zigbee 网关、MQTT、业务服务器、微信小程序的端到端架构。

可写结构：研究背景 -> 系统需求 -> 分层架构 -> 网关注册与状态归一化 -> 多用户绑定 -> 实验与延迟分析 -> 结论。

建议实验：连接稳定性、控制往返时间、并发设备数、断线重连、状态一致性。

## 方向二：基于 MCP 的自然语言智能家居控制方法

重点：将大模型工具调用与既有 IoT 业务服务安全连接，而不是让 AI 直接持有 MQTT 凭据。

可写结构：问题定义 -> MCP 工具设计 -> 内外网安全边界 -> 意图到设备能力映射 -> 状态确认机制 -> 准确率和可用性评估。

建议实验：查询正确率、控制成功率、字段完整率、歧义指令处理、失败提示真实性。

## 方向三：工程实践文章系列

### 文章 1：从 MQTT Demo 到可用微信智能家居

记录微信小程序 MQTT 兼容、合法域名、登录、网关绑定和设备卡片重构。

### 文章 2：网关在线但业务不通：MQTT 注册 ACK 排障实录

以红灯常亮、绿灯不亮为案例，说明连接层、注册层、状态层和控制层的区别。

### 文章 3：让小智 AI 安全控制 Zigbee 设备

介绍 Node 内部 API、Python MCP Bridge、共享密钥、工具设计和状态确认。

## 证据准备清单

- 系统架构图和消息时序图。
- 设备类型、测试数量与固件/协议版本表。
- 关键 Topic 与脱敏 JSON 示例。
- 控制延迟、成功率、在线时长和异常恢复记录。
- 小程序及语音交互截图。
- 失败案例和修复前后对照。
- 数据与代码可用性声明、隐私与安全说明。
