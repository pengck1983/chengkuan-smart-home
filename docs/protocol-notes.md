# MQTT 网关协议实现笔记

## Topic 模式

业务服务器按网关 MAC 订阅注册、子设备同步、设备状态、控制响应和事件 Topic。具体名称以设备厂商协议为准，代码中的通配订阅用于兼容多网关。

## 注册应答

网关发布 `register` 后，服务器必须向对应 `register_rsp` 返回相同 `sequence` 和成功错误码。缺少应答时，MQTT 可能仍在线、控制也可能偶尔有效，但网关业务指示灯会显示注册异常。

## 子设备同步

`add_subdevice` 和 `report_subdevice` 消息需要逐项返回同步结果。业务服务器据此创建或刷新设备记录，并根据 UIID 判断设备类型。

## 状态归一化

设备字段存在类型与倍率差异，归一化层需要：

- 保留原始消息，便于协议排查。
- 将字符串数值安全转为数字。
- 对温度、湿度、照度、电压、电流、功率、电量分别处理倍率。
- 区分零值与无数据，不能用 `value || default` 吞掉合法的 `0`。
- 用最新时间戳判断在线，不仅依赖单一 `online` 字段。

## 控制与确认

控制命令至少包含 `sequence`、`deviceId` 和 `params`。发布成功只表示消息进入 MQTT，不表示设备已执行。可靠交互应等待控制响应或新的设备状态，并在超时后提示“指令已发送，未收到设备确认”。

## 设备类别

- 计量插座：开关、电压、电流、功率、电量。
- 环境传感器：温度、湿度、照度、电池。
- 智能彩灯：开关、亮度、彩色模式、色温模式。
- 语音喇叭：文本转十六进制编码后下发 TTS，具体编码和警示音参数以设备协议为准。
